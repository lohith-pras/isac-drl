# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

DRL for beamforming in mmWave ISAC (Integrated Sensing and Communication) MIMO systems. A DDPG agent learns a continuous beamforming vector that trades off communication rate against sensing accuracy. University project (Projektarbeit, IDC chair, FAU). Roadmap is 5 phases (P1–P5); currently at end of P1. See `.hermes/plans/` for the phased plan and `docs/architecture.md`.

## Commands

```bash
uv sync                                          # install deps
uv run python -m pytest -v                       # run test suite (pytest is NOT on PATH; use `python -m`)
uv run python -m pytest tests/test_reward.py::test_sensing_reward_aligned   # single test
uv run python -m ruff check .                    # lint (ruff also not on PATH)
uv run python training/classical_baseline.py     # DFT steering-vector baseline
uv run python training/train_ddpg.py --timesteps 300000   # train DDPG
uv run python evaluation/evaluate_agent.py       # eval trained agent
uv run python evaluation/plot_results.py         # reward curve, scatter, beampattern
uv run python evaluation/pareto_curve.py         # comm-vs-sensing Pareto frontier
```

**Dependency gotcha:** `pyproject.toml` has both `[project.optional-dependencies].dev` (pytest, ruff) and `[dependency-groups].dev` (graphifyy). `uv sync` installs the dependency-group by default, so pytest/ruff land **not** in the env — invoke them as `uv run python -m pytest` / `-m ruff`, or `uv sync --extra dev`.

## Architecture

The pipeline is composition-based: four plain classes are constructed and injected into the Gym env. No global config object — every script (`make_env`/`make_isac_env`) hand-builds the stack with literal params (`Nt=4, Nr=4`, `28e9` Hz, `speed_kmh=60`, `distance=100`). Changing system params means editing each factory.

**Construction order** (env factory in any train/eval script):
`MIMOSystem` → `V2XScenario` → `SVChannelModel(mimo=...)` → `RewardConfig` → `ISACEnv(mimo, scenario, channel, reward_config)`.

- **`environment/mimo_system.py`** — ULA physics. `tx/rx_steering_vector(theta_deg)`, `array_gain(w, theta)`. Owns `_wavelength` (used by the channel model for path loss + Doppler).
- **`environment/channel_model.py`** — `SVChannelModel`: Saleh-Valenzuela clustered channel. `generate(velocity_ms, distance)` returns the `(Nr, Nt)` matrix `H` with small-scale fading + free-space path loss + optional Doppler. `coherent=True` freezes cluster/ray geometry at `reset_coherent()` and only evolves Doppler phase per step (smooth motion); default mode resamples ray angles every call (jittery).
- **`environment/v2x_scenario.py`** — 1D vehicle motion. `step()`/`reset()` return `{distance, velocity, angle_of_arrival}`; AoA/distance are recomputed from geometry each step.
- **`environment/isac_env.py`** — the hub. Action = flat `2*Nt` real vector in `[-1,1]` → mapped to a unit-norm complex `w` scaled to `sqrt(max_tx_power)` (`_action_to_beamformer`). Observation = `[H.real, H.imag, distance, velocity, aoa]`. On `step()` it computes reward, then advances scenario and regenerates `H`.

**Reward** (`alpha * comm_norm + beta * sens_norm`, both in [0,1], weights in `utils/reward_config.py`):
- Comm: `log2(1 + ||Hw||²/σ²)` normalized by a fixed Shannon bound computed once in `__init__`.
- Sensing: squared cosine similarity `|aᴴw|² / (||w||²·Nt)` — normalized by construction, **no runtime calibration**. An older random-action calibration path (`_max_sensing_gain`) was removed in P1; the bound is now simply 1.0.

Power values use `dbm_to_linear` (returns mW despite "Watts" comments) — consistent across signal and noise, so SNR is correct.

**Training stack:** `Monitor → DummyVecEnv → VecNormalize(norm_obs=True, norm_reward=False)`, SB3 `DDPG` + Ornstein-Uhlenbeck noise with a linear-decay callback. A trained model **must** be loaded with its saved `VecNormalize` stats or observations are wrong.

**Artifacts:** training saves the final agent + its VecNormalize stats to `models/ddpg_isac_final/{model.zip, vecnormalize.pkl}`; `EvalCallback` also drops a `best_model.zip` in `models/ddpg_isac_best/` (no vecnormalize there). Eval and plot scripts load from `ddpg_isac_final/` — keep them and training pointed at the same dir. All model/`.npy`/tensorboard artifacts are gitignored; the committed result figures live in `docs/figures/` and are regenerated by `evaluation/plot_results.py` + `pareto_curve.py`.

## Conventions

- Steering-vector angles are in **degrees** at the public API; converted to radians internally.
- Channel matrix shape is `(Nr, Nt)`; beamformer `w` is shape `(Nt,)`.
- ruff: line-length 100, rules `E,F,I,W`; excludes `notebooks/`, `scratch/`, `graphify-out/`, `.agents/`.
